# Стратегія тестування

- Всі тести знаходяться в папці `tests/`.
- Використовуй `pytest`.
- Покриття має бути > 70%.
- Заборонено тестувати UI (Flet), тестуємо тільки бізнес-логіку (`services`) та репозиторії.
- Використовуй `pytest-mock` для ізоляції тестів від файлової системи.
- Кожен коміт повинен запускати `pytest --cov=src --cov-report=xml`.
